# `neo4j.name`

## Client need

Multi-release Neo4j cluster installs require a stable logical cluster name shared across Helm releases so discovery, load balancers, and label selectors identify the same Neo4j deployment. Every release in a cluster must set the same `neo4j.name` while using distinct release names.

## Neo4j documentation

- [Clustering — discovery](https://neo4j.com/docs/operations-manual/current/clustering/) — member identification
- [Neo4j on Kubernetes](https://neo4j.com/docs/operations-manual/current/kubernetes/) — multi-release cluster pattern

## Helm implementation

- **Templates**: `helm-charts/neo4j/templates/_helpers.tpl` (`neo4j.name` — required non-empty); used in labels `app`, `helm.neo4j.com/neo4j.name`, ConfigMap `dbms.kubernetes.label_selector`, `neo4j-loadbalancer.yaml` collision check, env `K8S_NEO4J_NAME`
- **Go model**: `release_values.go` — `Neo4J` name field (via struct)
- **K8s resources**: StatefulSet, Services, ConfigMaps, Secrets, PDB — label selector `app={{ neo4j.name }}`
- **Neo4j mechanism**: Kubernetes discovery label selector ties cluster members together
